Weekly Blog #2

Jan 18 - Feb 3

Construction is underway and the team is working hard to get a working prototype. Lots of updates on progress and design changes as well as plans to receive input from the hockey community on the thoughts of our idea. This is a continuation of the previous update blog so reading that before continuing is highly recommended.

Website is Live

The AfterShock website is now officially live. The website features description of the product, testimonials from industry specialists and a blog page featuring weekly updates and technical blogs detailing different technology used throughout the project. The website can be reached at after-shock.ca

Software

There have been a few updates to the software since the last update. The REST API and react-native application have been combined to create a user login and registration page for the mobile app. The user login is to protect the data the application will contain as well as assign user roles so each user has the data tailored to what their role needs to view.
On the firmware side the network communication between the head mounted device and the REST API have been complete. Data from the device can be successfully sent to an API endpoint which stores it in our database. The firmware is also capable of allowing the server to send messages to the device, this will allow us to remotely control some features on the device.

Electrical

On the electrical side the big update is the new arduino nano has arrived and development with the board has begun. Attempted integration with the original accelerometers specified proved to not be possible given the technology we are working with. For this reason new accelerometers have been specified and prototyping has begun.
Upcoming tasks on the electrical side include successfull communication with the sensors, signal processing and filtering from the multiple sensors on the device and design a wire layout. The wire layout is crucial for further development as it impacts the design of the cap itself as well as the casing the board will rest in.

Mechanical

The 3D enclosure to contain the communication board has changed many times as prototyping has continued. The current design is a hard enclosure that has very narrow dimensions to allow it to be kept inside of the device itself allow us to have a self-contained unit.
A new champion has been decided for the design of the cap the will contain the sensors. Cameron Blair has taken the initiative towards the completion of the project and will oversee the tasks involved in completing the design.

Testing

Cameron Blair has met with Professor Duane Cronin to discuss methods for testing as well as a better understanding behind concussion causes and diagnosis. Dr. Cronin has a doctorate of Mechnical Engineering and specializes in analyzing head trauma in automotive accidents and in football. He has shared with us the most up to date models for detecting concussions. He has also given us testing suggestion of wearing the device during normal daily routine to help eliminate false positives.
The team also plans on testing the device in a game scenario once the prototype is complete. The testing is tentative on the timeline of completion.

Project Management

After a few rounds of redesign the project is finally back on track. The planned completion is March 6, 2019 with incremental milestones between then and now. The project is also coming in under budget and has lots of flexibility in the case of an unpredicted problems.

Eric Tweedle

Project Manager / Front-End Software Lead
