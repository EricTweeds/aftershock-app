Weekly Blog #3

Feb 4 - Feb 17

The team members have been working on their individual components of the project and how the components will connect together to form the completed product. This is a continuation of the previous update blog so reading that before continuing is highly recommended.

Software

The work done on the mobile application has been designing different views to ensure that they are user-friendly and easy to understand. This is an important portion of the project as making the data insights easy to understand and act upon is the major selling point of the product. Once all the different views have been designed and build they can be fully integrated with the REST API.

The REST API has been updated with new endpoints to support the functionality offered by the mobile application, such as push notifications. Since the communication between the head-mounted device and the REST API has been complete, the next step is to implement the algorithms which will derive insights from the player data.

Electrical

Communication between the Arduino and the new accelerometers has successfully been implemented and tested. This data can now be used to test the data analysis algorithms. Cameron D the head of electrical is working with Cameron B to determine the most optimal layout for electrical wiring throughout the cap.

Mechanical

The enclosure for the communication board has been 3D printed. After playing with the enclosure it was noticed that the opening for the electrical wires was too small. Also, the current design of the enclosure makes it difficult to reach the Arduino inside it. With these new considerations, another enclosure is being designed. The reason the enclosure was printed early was exactly for the reason of being able to quickly iterate upon the design.

Project Management

As the deadline for the project approaches the team has determined a more detailed timeline for the remainder of the project. This will ensure that all team members are on the same page and that the project is completed with a sufficient amount of time to perform testing and final adjustments.

Bradley McAllister

Backend Software Lead
