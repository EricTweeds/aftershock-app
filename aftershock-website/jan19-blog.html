Say Hello

Weekly Blog #1

Jan 6 - Jan 17

Our team is back after the winter break to start creating the first prototype of the concussion monitoring device. Each member has begun working on different parts of the project.

MTE 482

For the FYDP class Eric has build the skeleton for the teams website. The team is working to fill in information on the site.

Software

The software side of the project includes the firmware on the monitoring device, a REST API to receive and analyze the acceleration data and the mobile app which displays the data and results. The electrical board being used is being shipped, so work on the firmware has not yet begun. The REST API is being developed by Bradley and currently has all the required endpoints to get data and results. The API requires more work to be able to analyze the data. Eric has begun working on the mobile app. He has developed wireframes which show how the data will be displayed and interacted with the end user. The mobile app is also using the REST API to begin fetching data.

Electrical

The electrical portion of the project is very important since the data analysis, mechanical enclosures and testing methods depend on the main board and accelerometers to be setup. The accelerometers are small so boards are required for each accelerometers. Cameron D has designed and printed PCB’s for the accelerometers.

The main board was originally meant to be a PCB, but due to shipping times and board uncertainty the team decided to use an Arduino Nano IOT. The Arduino will simplify much of the firmware development. The Arduino also has a built in wifi module which will allow the board to easily transmit data over wifi.

Mechanical

Marcus has begun designing 3D enclosures for each of the accelerometers and the main board. The enclosures are planned on being 3D printed. The enclosures will need to be combined with the material which will hold the accelerometers in place on the players head. The enclosures are blocked from being finished by the electrical components. The wiring and boards need to be finalized and soldered before the enclosures can be finished. Some material has been chosen and will be tested once the enclosures have been created.

Testing

Cameron B has begun investigating methods for how the device will be tested. Testing is very important for the success of the project since it will allow the group to collect sample acceleration data and validate the analysis of the data. Cameron B met with Professor Michael Worswick who works at the Crash Lab. Cameron B and the Professor talked about a sled in the lab which can reach 100g’s. A drop test might be able to allow the group to reach a higher acceleration value.

The project is on schedule right now. The team is excited for the board to get in so the firmware can begin!

Bradley McAllister
